Data Engineering · Visualization

Engine Analytics Dashboard

Carnival Cruise Line's engine data lived in 54+ inconsistent Excel files with no unified view, making trend analysis nearly impossible.

Built a Power BI dashboard that standardized and visualized engine KPIs across the fleet. Cleaned all 54 datasets, resolved schema conflicts, and delivered a single source of truth adopted by the ops team.

Chose Power BI over Python dashboards because end users were operations staff, not analysts — adoption risk mattered more than technical flexibility. Normalized schemas manually before scripting, which surfaced data entry patterns a script alone wouldn't have caught.
Power BIExcelData CleaningKPI Design

Systems · Infrastructure

Virtualized Active Directory Lab

Needed to understand enterprise IT architecture without access to enterprise infrastructure.

Designed and deployed a multi-site Active Directory environment on Windows Server — replication, domain services, group policies, and automation. Simulated real-world IT complexity on local hardware.

Built multi-site replication rather than a single domain because real enterprise environments almost always span locations. Wrote PowerShell automation after realizing manual provisioning wouldn't scale even in a lab context.

Process Improvement · Analytics

Lean Six Sigma Demand Analysis

Multi-year ticket sales data existed but no one had analyzed seasonal patterns or where waste was accumulating.

Applied DMAIC to surface demand trends and quantify opportunity areas. Built in SQL with recommendations tied to specific process steps — not a generic report.

Used DMAIC instead of ad-hoc analysis because stakeholders needed a repeatable process they could run next quarter themselves. Focused the Improve phase on two high-confidence changes rather than listing every possibility — kept it actionable.

Python · Tableau · Data Analysis

International Travel Cost Optimization

A group of 7 people needed to decide between three international trip options — no one had modeled the actual costs.

Analyzed 30 years of US flight fare data. Identified Chicago → Orlando at $2,154 for 7 people and Q3 as optimal booking window. Built an interactive Tableau dashboard.

Started with 3 hardcoded rows — expanded into a real analysis using a Kaggle dataset of 245,000+ records from 1993–2024. Added cost-per-day and group total because raw fare prices alone were misleading.
