Projects

Problems I chose to solve, decisions I made, and what came out of it.

Data Engineering · Visualization
Engine Analytics Dashboard

Problem: Carnival Cruise Line's engine data lived in 54+ inconsistent Excel files with no unified view, making trend analysis nearly impossible.

Built a Power BI dashboard that standardized and visualized engine KPIs across the fleet. Cleaned all 54 datasets, resolved schema conflicts, and delivered a single source of truth adopted by the ops team.

Chose Power BI over Python dashboards because end users were operations staff, not analysts — adoption risk mattered more than technical flexibility. Normalized schemas manually before scripting, which surfaced data entry patterns a script alone wouldn't have caught.
Power BIExcelData CleaningKPI Design
Systems · Infrastructure
Virtualized Active Directory Lab

Problem: Needed to understand enterprise IT architecture without access to enterprise infrastructure.

Designed and deployed a multi-site Active Directory environment on Windows Server — replication, domain services, group policies, and automation. Simulated real-world IT complexity on local hardware.

Built multi-site replication rather than a single domain because real enterprise environments almost always span locations. Wrote PowerShell automation after realizing manual provisioning wouldn't scale even in a lab context.
Windows ServerActive DirectoryPowerShellNetworking
Process Improvement · Analytics
Lean Six Sigma Demand Analysis

Problem: Multi-year ticket sales data existed but no one had analyzed seasonal patterns or where waste was accumulating in the pipeline.

Applied DMAIC to surface demand trends, identify inefficiencies, and quantify opportunity areas. Built the analysis in SQL with recommendations tied to specific process steps — not a generic report.

Used DMAIC instead of ad-hoc analysis because stakeholders needed a repeatable process they could run next quarter themselves. Focused the Improve phase on two high-confidence changes rather than listing every possibility — kept it actionable.
SQLLean Six SigmaDMAICData Analysis
Python · Data Analysis · Visualization
International Travel Cost Optimization

Problem: A group of 7 people needed to decide between three international trip options from Mongolia to the U.S. — no one had modeled the actual costs.

Built a Python analysis comparing total cost, cost per person, and cost per day across three itinerary options. Visualized all scenarios with bar charts and identified the optimal route programmatically — turning a group debate into a data-backed decision.

Added cost-per-day as a metric because total cost alone was misleading — a longer trip costs more but might actually be cheaper per day. Used pandas to structure the comparison so anyone in the group could read the output without knowing Python. This was a real decision for real people, not a classroom exercise.
