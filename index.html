Engine Analytics Dashboard

Problem: Carnival Cruise Line's engine data lived in 54+ inconsistent Excel files with no unified view.

Built a Power BI dashboard that standardized and visualized engine KPIs across the fleet. Delivered a single source of truth adopted by the ops team.

Chose Power BI over Python dashboards because end users were operations staff, not analysts — adoption risk mattered more than technical flexibility. Normalized schemas manually before scripting, which surfaced data entry patterns a script alone wouldn't have caught.

Career Tracker — AI Job Application Dashboard

Problem: 90+ job applications across different stages with no system to track them, generate tailored documents, or practice interviews.

Built and deployed a full-stack web app with a kanban board, AI cover letter generator, mock interview chat, and analytics — in 5 days. Live and in active use.

Chose a serverless architecture to keep the Anthropic API key secure — the key lives on Netlify's servers, never exposed in the frontend. Python · Tableau · Data AnalysisApr 2026
International Travel Cost Optimization

Problem: A group of 7 people needed to decide between trip options — no one had modeled the actual costs.

Analyzed 4,500+ rows of flight fare data. Identified Chicago → Orlando as the cheapest route at $2,154 for 7 people — 32% cheaper than Chicago to LA. Built an interactive Tableau dashboard.

Started with 3 hardcoded rows — expanded into a real analysis using a Kaggle dataset of 245,000+ flight records from 1993–2024. Added cost-per-day and group total as metrics because raw fare prices alone were misleading.

Lean Six Sigma Demand Analysis

Problem: Multi-year ticket sales data existed but no one had analyzed seasonal patterns or where waste was accumulating.

Applied DMAIC to surface demand trends, identify inefficiencies, and quantify opportunity areas. Recommendations tied to specific process steps — not a generic report.

Used DMAIC because stakeholders needed a repeatable process they could run next quarter themselves. Focused the Improve phase on two high-confidence changes rather than listing every possibility — kept it actionable.
